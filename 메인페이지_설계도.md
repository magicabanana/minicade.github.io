# 📦 TOYBOX 메인 페이지 설계도

## 1. 프로젝트 개요
- **목적**: 다양한 미니게임과 인터랙티브 도구들(사과게임, 이재용물가체감기 등)을 한곳에 모아두는 허브/포털 페이지
- **디자인 모티브**: Neal.fun 스타일의 깔끔하고 직관적인 미니게임 모음집 UI
- **특징**: 자바스크립트를 배제하고 100% 순수 CSS(CSS Grid, Transform)만으로 애니메이션 및 반응형 레이아웃을 완전히 구현한 정적 페이지

---

## 2. 파일 구조 및 역할

- `index.html`: SEO 최적화 메타데이터 포함, 메인 레이아웃 뼈대, 개별 미니게임으로 연결되는 카드 컴포넌트 마크업
- `style.css`: CSS Variables(`--bg-color` 등)를 활용한 테마 기반. 반응형 그리드 시스템 및 3D 입체 호버 애니메이션 지정
- `script.js`: 내용 없음 (기존 JS 호버 로직을 모두 CSS로 이관함)

---

## 3. UI/UX 디자인 (style.css)

### 🎨 테마 및 타이포그래피
- **가독성 확보**: 미니멀리즘 컬러 (`--bg-color`, `--text-color`)
- **Fonts**: 타이틀 헤딩용 `Outfit`, 범용 본문용 `Inter` 서체 적용

### 📐 주요 레이아웃 구조
1. **Header (`<header>`)**: 
   - 스크린리더 전용 은닉 h1 (`.sr-only`) 요소를 포함하여 접근성과 SEO 최적화
   - 픽셀이 깨지지 않게 렌더링 옵션(`image-rendering`)이 적용된 메인 로고(`title.png`)
2. **Main / Grid (`div.grid`)**:
   - `CSS Grid`를 활용하여 화면 너비에 따라 열 개수가 변하는 적응형 구조 설계:
     - 기본(PC): 가로 3열 (`grid-template-columns: repeat(3, 1fr)`)
     - 태블릿(`max-width: 900px`): 가로 2열
     - 모바일(`max-width: 600px`): 가로 1열
3. **Footer (`<footer>`)**:
   - 사이트 하단에 개발자 정보, 소셜 연결 버튼 모음, 연락처 등 배치

### ✨ 애니메이션 및 효과
- **100% CSS 트랜지션**: 마우스 오버(Hover) 시 `.card:hover`를 통해 부드러운 스케일 업 애니메이션 구현 (`transform: scale(1.05)`)
- **3D 입체 호버 효과**: 
  - 그리드 컨테이너에 `perspective: 1000px`을 주어 원근감을 형성하고,
  - 카드 자체를 `transform-style: preserve-3d`로 두어 `box-shadow`와 함께 화면 앞으로 떠오르는 깔끔한 호버(Hover) 동작을 구현
- **글래스모피즘 그라데이션 Fallback**: 배경 이미지가 삽입되지 않은(.card-image) 카드를 위해 1~6번째 각각의 자식 요소(`nth-child`)에 부드러운 파스텔톤 그라데이션 선계를 미리 지정해 안정적인 디자인을 유지

---

## 4. SEO 및 최적화 특징 (`index.html`)
- **Open Graph / Twitter Card 적용**: 소셜 미디어(카카오톡, 트위터 등)으로 공유될 때 표시되는 미리보기 최적화 (`og:image`, `twitter:card` 등 완비)
- **JSON-LD 구조화 데이터 삽입**: 검색 엔진(Google)에게 이 사이트가 "TOYBOX"라는 웹사이트(`WebSite`)이며 내부적으로 10여개의 미니게임 리스트(`ItemList`)를 보유하고 있음을 명시적으로 알려 크롤링을 도움
- **구글 애드센스 적용**: 페이지 수익화를 돕는 비동기 애드센스 호출 스크립트 포함
